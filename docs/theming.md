# Theming skylobby

The canonical reference for theme authors. It enumerates every selector,
colour-ramp key, and skylobby-specific style class you can target, so you can
reskin skylobby without reverse-engineering the source.

Everything below is derived from `src/clj/skylobby/fx.clj` (the definitions are
the source of truth — see [Maintenance](#maintenance)). Selector names are quoted
exactly as they appear in code; a wrong selector name is the most common reason a
theme tweak silently does nothing.

## Overview

skylobby's UI is built with [cljfx](https://github.com/cljfx/cljfx) over JavaFX,
and styled with [cljfx/css](https://github.com/cljfx/css). A theme is a plain
data map — a **style-data map** — of the form:

```clojure
{"selector" {:-fx-property value
             "-nested-suffix" {:-fx-property value}}}
```

Keys are JavaFX CSS selector strings; values are maps of JavaFX CSS properties
(`:-fx-*`). A nested map concatenates its key onto the parent selector, so

```clojure
{".skylobby" {"-chat" {"-message" {:-fx-fill "white"}}}}
```

compiles to a rule for `.skylobby-chat-message`. This is how the large chat/IRC
colour set is expressed compactly.

### Where styles come from, and the cascade

`stylesheet-urls` (fx.clj) loads exactly two stylesheets, in this order:

1. **`default-classes-css`** — the theme-independent skylobby classes
   (typography, control sizing, form-label, chat/console/IRC colours, …). Always
   loaded first.
2. **The active theme** — either a shipped preset or your custom CSS.

JavaFX applies later stylesheets over earlier ones, and within a stylesheet the
JavaFX cascade/specificity rules apply on top of the framework's own
`modena`/`caspian` base.

**The golden rule:** your custom CSS is appended **last**, so it always wins over
both the framework defaults and `default-classes`. Anything you put in
`custom-css.edn` is the final word. (When a preset is active instead of custom
CSS, the preset occupies that last slot.)

> A preset's style-data is produced by `theme-data` from a colour ramp and then
> merged with a few per-theme extras — so a preset already *contains* the
> resolved selector rules. Custom CSS is just a style-data map you author by hand
> in the same shape.

## How to apply a theme

### The file

Custom CSS lives in **`custom-css.edn`** in the skylobby **app root** (the
per-user data directory where `settings.edn`, logs, etc. live — the exact path is
shown in Settings, see below). On first run skylobby seeds this file with a
pretty-printed copy of the default (black) theme's style-data, so you always have
a complete, working starting point to edit.

The file content is a single EDN **style-data map** — the same shape described in
[Overview](#overview):

```clojure
{".root" {:-fx-base "rgb(30,30,30)"
          :-fx-background "rgb(18,18,18)"}
 ".skylobby-primary" {:-fx-base "#3aa0ff"
                      :-fx-background "#3aa0ff"}}
```

### Loading and hot-reload

Open **Settings → Appearance**. There you'll find:

- Five **preset buttons**: Black (default), Grey, Light, Catppuccin Mocha,
  Catppuccin Latte.
- A **"Load custom CSS as EDN from: `<path>`"** button. The path shown is your
  `custom-css.edn`. Clicking it parses the file and applies it; a success/error
  message appears beneath.

Once custom CSS has been loaded (or any time the app is running), **saving
`custom-css.edn` hot-reloads it automatically** — a background watcher
(`css-reload-chimer-fn`, polling once per second) notices the file's modified
time changing and re-applies it. Edit, save, watch the UI update. No restart.

### How custom CSS is registered (and why image paths work)

When CSS is applied, `file-backed-css` compiles your style-data and **writes the
compiled CSS to a file** (`skylobby-current.css`) in the app root, then points
JavaFX at that file. Because JavaFX resolves relative `url(...)` references
against the stylesheet's own location, this means you can reference image assets
(e.g. a welcome background) by paths **relative to the app root**, dropped
alongside `custom-css.edn`. A content-hash fragment is appended to the URL so
hot-reload reliably re-triggers even when the path is unchanged.

## Two ways to theme

There are two distinct levels you can work at. They are not mutually exclusive.

### (a) Re-tint by overriding the ramp *(conceptual / source-level)*

Every shipped dark/light theme is generated by feeding a **colour ramp** — a flat
map of ~18 named colours (`surface-0`, `border`, `focus`, …) — to the
`theme-data` function, which projects those colours onto all the JavaFX
selectors. This is how Catppuccin Mocha was built: copy a ramp, swap the colour
values, done. See [Colour-ramp reference](#colour-ramp-reference).

This approach lives in `fx.clj` (you copy `*-ramp` and call `theme-data`). From
`custom-css.edn` you cannot call `theme-data`, but you can reproduce its effect:
the most important ramp colours surface as JavaFX **looked-up colours** on
`.root` (`-fx-base`, `-fx-background`, `-fx-accent`, …). Override those few
looked-up colours and most of the UI re-tints automatically, because nearly every
control derives its colour from them.

**Use when:** you want a coherent whole-app recolour with minimal effort.

### (b) Override individual selectors in `custom-css.edn`

Target specific selectors directly — restyle just the primary button, just the
chat highlight colour, just one screen. Because custom CSS wins last, any rule you
write here overrides the preset.

**Use when:** you want surgical changes, or to layer accents on top of a preset.

The common pattern is to combine both: set the looked-up colours on `.root` for a
base re-tint, then override a handful of specific selectors for accents.

## Colour-ramp reference

A ramp is a flat map consumed by `theme-data`. The destructured argument list in
`theme-data` is the authoritative key set. Each key drives the JavaFX properties
listed below.

| Ramp key | Drives |
|---|---|
| `surface-0` | App background. `.root` `-fx-background`; `.tab:selected` background; `.tab-header-background`; `.styled-text-area` background. The deepest surface. |
| `surface-1` | `.root` `-fx-base` (the framework's master control colour); `.tab` background; `.table-view` background. |
| `surface-2` | `.root` `-fx-control-inner-background`; `.tab` base; card background (`.skylobby-card`); `.skylobby-secondary:hover`; `.skylobby-primary:disabled` fallback surface; table column-header background. |
| `surface-3` | `.root` `-fx-accent` (selection/highlight accent); slider track. The raised/hover/selected surface. |
| `border` | Card border; separator lines; table border + cell border lines. |
| `focus` | `.root` `-fx-focus-color` (focus ring); slider thumb on hover/press. |
| `selection` | Selected table-row background (focused). |
| `selection-unfocused` | `.root` `-fx-selection-bar-non-focused` (selected rows when the table isn't focused). |
| `text-on-dark` | Light end of the `.root` text-colour `ladder(...)` (used when the surface is dark). |
| `text-on-light` | Dark end of the `.root` text-colour `ladder(...)` (used when the surface is light). |
| `text-2` | Secondary/muted text: text-field prompt text; combo-box/scroll-bar arrows. |
| `row-odd` | Odd table-row stripe. |
| `row-even` | Even table-row stripe. |
| `thumb` | Scroll-bar thumb base colour. |
| `thumb-hover` | Scroll-bar thumb on hover; slider thumb (resting). |
| `tab-highlight` | The focused-tab highlight (consumed by per-theme `.skylobby-tab-focus` extras, e.g. gold). |
| `tab-selected-accent` | `.tab:selected` `-fx-accent`. |
| `icon` | `.ikonli-font-icon` glyph colour (all icons). |

### Looked-up colours you can reference

`theme-data` sets these JavaFX **looked-up colours** on `.root`. Override them
from `custom-css.edn` for a broad re-tint; reference them in your own rules with
the property value e.g. `:-fx-text-fill "-fx-text-base-color"`.

| Looked-up colour | Set from | Meaning |
|---|---|---|
| `-fx-base` | `surface-1` | JavaFX master colour most controls derive from. |
| `-fx-background` | `surface-0` | Window/region background. |
| `-fx-accent` | `surface-3` | Selection/highlight accent. |
| `-fx-control-inner-background` | `surface-2` | Inner background of inputs/lists/tables. |
| `-fx-selection-bar-non-focused` | `selection-unfocused` | Selected rows in unfocused controls. |
| `-fx-focus-color` | `focus` | Focus ring colour. |
| `-fx-faint-focus-color` | `transparent` | Outer faint focus glow (disabled). |
| `-fx-text-base-color` | `ladder(-fx-base, text-on-dark 49%, text-on-light 50%)` | Auto-contrast text colour. |

**`ladder(...)` auto-contrast:** the text colour is chosen by measuring the
brightness of `-fx-base`. Below the 49–50% threshold (dark surface) it resolves to
`text-on-dark`; above it (light surface) it resolves to `text-on-light`. This is
why simply changing `-fx-base` flips text between light and dark automatically —
you usually don't need to set text colours by hand.

## Selector reference

"default-classes" = theme-independent (loaded first, overridable). Properties
listed are what skylobby sets — you can set any valid JavaFX CSS property.

wired into the generated CSS).

| Selector | Source | Size | What it is |
|---|---|---|---|
| `.skylobby-h1` | default-classes | 22px | Primary heading. |
| `.skylobby-h2` | default-classes | 18px | Secondary heading. |
| `.skylobby-body` | default-classes | 16px | Body text. |
| `.skylobby-caption` | default-classes | 13px | Small/caption text. |

## Worked examples

All examples are the full content (or a fragment to merge) of `custom-css.edn`.
Save the file and skylobby hot-reloads it.

### 1. Tweak one accent colour

Make the primary CTA blue instead of gold, leaving everything else as the default
theme:

```clojure
{".skylobby-primary" {:-fx-base "#3aa0ff"
                      :-fx-background "#3aa0ff"}}
```

> If you start from the seeded `custom-css.edn` (a full copy of the black theme),
> just edit the `.skylobby-primary` entry already in it rather than replacing the
> whole file.

### 2. Re-tint a single screen

Give the battle screen a faint warm wash without touching other screens:

```clojure
{".skylobby-screen-battle" {:-fx-background-color "rgb(34,28,28)"}}
```

### 3. Restyle the primary button (resting + disabled)

```clojure
{".skylobby-primary"          {:-fx-base "#2ecc71"
                               :-fx-background "#2ecc71"}
 ".skylobby-primary:disabled" {:-fx-base "rgb(40,40,40)"
                               :-fx-background "rgb(40,40,40)"}}
```

### 4. Broad re-tint via looked-up colours (the Catppuccin approach)

The shipped presets are built by handing a colour ramp to `theme-data`. You can
reproduce most of that effect from `custom-css.edn` by overriding the looked-up
colours on `.root` — most controls re-derive their colour automatically, and text
contrast flips via `ladder(...)`:

```clojure
{".root" {:-fx-base "rgb(30,30,46)"          ; ~ catppuccin base  → surface-1
          :-fx-background "rgb(24,24,37)"     ; mantle             → surface-0
          :-fx-accent "rgb(69,71,90)"         ; surface1           → surface-3
          :-fx-control-inner-background "rgb(49,50,68)"
          :-fx-focus-color "rgb(137,180,250)" ; blue}
 ".ikonli-font-icon" {:-fx-icon-color "rgb(205,214,244)"}}
```

To build a *complete*, polished theme (every table stripe, scroll-bar gradient,
etc.) the cleanest path is still the ramp approach in `fx.clj`: copy
`catppuccin-mocha-ramp`, change the colour values, and feed it to `theme-data`
(plus optional per-theme extras). See [Maintenance](#maintenance).

## Maintenance

This document is a hand-written reference. The **source of truth** is
`src/clj/skylobby/fx.clj`:

- `black-ramp` / `grey-ramp` / `light-ramp` / `catppuccin-mocha-ramp` — ramp keys.
- `theme-data` — maps a ramp onto the JavaFX selectors. Its destructured argument
  list **is** the ramp-key contract.
- `grey-style-data` / `black-style-data` / `light-style-data` /
  `catppuccin-mocha-style-data` — per-theme extras merged over `theme-data`.
- `default-classes` — the theme-independent skylobby classes (typography, control
  sizing, form-label, chat/console/IRC/spring-log colours, player classes).
- `style-presets`, `default-style-data` — the shipped preset registry.
- `stylesheet-urls` — load order (default-classes first, active theme last).
- `file-backed-css` — how custom CSS is written to disk so relative image paths
  resolve.

Plumbing (custom-css.edn seeding, load handler, hot-reload watcher) lives in
`src/clj/spring_lobby.clj` (`::update-css`, `::load-custom-css-edn`,
`css-reload-chimer-fn`, and app-init). The user-facing entry is the "Appearance"
section in `src/clj/skylobby/fx/settings.clj`.

If you change a selector name or ramp key in code, update the matching row here.
